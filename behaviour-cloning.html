Behaviour Cloning

The most straightforward form of imitation learning.

Let's start with the most straightforward one: imitation learning. This technique is simple and yet quite powerful, especially for simpler tasks or smaller MLP architectures.

Main Concept

The main concept is exactly what its name sounds like: copying actions from expert behaviour. You show the robot how to do the task through teleoperation or another demonstration method. Then the model learns a policy that maps states or observations to the expert actions it saw in the demonstrations.

Loss Functions

Two common supervised loss functions for imitation learning are negative log likelihood and mean squared error.

Negative Log Likelihood

For classification, we use the negative log likelihood:

NLL(θ) = -1/N ∑i=1N log πθ(ai | si)

You can think of πθ(ai | si) as the probability that the policy assigns to the expert action ai given the state si.

Mean Squared Error

For continuous action outputs, we use the mean squared error:

LMSE(θ) = 1/N ∑i=1N ||ai - πθ(si)||2

You can think of it like the difference between the actual expert actions and the actions computed by the policy given the state.

import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
import mujoco
import imageio


XML = """
<mujoco>
  <option timestep="0.02" gravity="0 0 -9.81"/>

  <worldbody>
    <light pos="0 0 3"/>
    <camera name="cam" pos="1.2 -1.2 0.9" xyaxes="1 1 0 -0.4 0.4 1"/>

    <geom name="table" type="box" pos="0 0 0" size="0.6 0.4 0.03" rgba="0.6 0.6 0.6 1"/>

    <body name="cube" pos="-0.25 0 0.08">
      <joint name="cube_free" type="free"/>
      <geom type="box" size="0.04 0.04 0.04" rgba="0.1 0.4 1 1"/>
    </body>

    <body name="gripper" mocap="true" pos="-0.25 0 0.25">
      <geom type="sphere" size="0.035" rgba="1 0.2 0.2 1"/>
    </body>

    <geom name="target" type="cylinder" pos="0.25 0 0.04" size="0.06 0.005" rgba="0.2 1 0.2 1"/>
  </worldbody>
</mujoco>
"""


class Policy(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(9, 64),
            nn.ReLU(),
            nn.Linear(64, 64),
            nn.ReLU(),
            nn.Linear(64, 3)
        )

    def forward(self, x):
        return self.net(x)


def expert_action(gripper, cube, target):
    above_cube = cube + np.array([0, 0, 0.18])
    grasp_pos = cube + np.array([0, 0, 0.06])
    above_target = target + np.array([0, 0, 0.18])
    place_pos = target + np.array([0, 0, 0.08])

    dist_to_cube = np.linalg.norm(gripper[:2] - cube[:2])
    holding = np.linalg.norm(gripper - grasp_pos) < 0.06 or cube[2] > 0.12

    if not holding:
        goal = above_cube if dist_to_cube > 0.03 else grasp_pos
    else:
        goal = above_target if np.linalg.norm(gripper[:2] - target[:2]) > 0.03 else place_pos

    action = goal - gripper
    return np.clip(action, -0.03, 0.03)


def get_state(gripper, cube, target):
    return np.concatenate([gripper, cube, target]).astype(np.float32)


def set_cube_pos(data, pos):
    data.qpos[:3] = pos
    data.qpos[3:7] = np.array([1, 0, 0, 0])


model = mujoco.MjModel.from_xml_string(XML)
data = mujoco.MjData(model)

target = np.array([0.25, 0.0, 0.08])

states = []
actions = []

# Generate expert demonstrations
for episode in range(80):
    cube = np.array([
        np.random.uniform(-0.35, -0.15),
        np.random.uniform(-0.12, 0.12),
        0.08
    ])
    gripper = cube + np.array([0, 0, 0.22])
    holding = False

    for t in range(120):
        action = expert_action(gripper, cube, target)

        states.append(get_state(gripper, cube, target))
        actions.append(action.astype(np.float32))

        gripper = gripper + action

        if np.linalg.norm(gripper - (cube + np.array([0, 0, 0.06]))) < 0.06:
            holding = True

        if holding:
            cube = gripper - np.array([0, 0, 0.06])

        if holding and np.linalg.norm(cube[:2] - target[:2]) < 0.04 and cube[2] < 0.12:
            holding = False

states = torch.tensor(np.array(states))
actions = torch.tensor(np.array(actions))

policy = Policy()
optimizer = optim.Adam(policy.parameters(), lr=1e-3)
loss_fn = nn.MSELoss()

# Behaviour cloning training
for epoch in range(200):
    pred = policy(states)
    loss = loss_fn(pred, actions)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if epoch % 50 == 0:
        print(f"epoch {epoch}, loss = {loss.item():.5f}")


# Roll out learned policy and render video
renderer = mujoco.Renderer(model, height=480, width=640)
frames = []

cube = np.array([-0.28, 0.08, 0.08])
gripper = cube + np.array([0, 0, 0.22])
holding = False

for t in range(160):
    state = torch.tensor(get_state(gripper, cube, target)).unsqueeze(0)

    with torch.no_grad():
        action = policy(state).numpy()[0]

    action = np.clip(action, -0.03, 0.03)
    gripper = gripper + action

    if np.linalg.norm(gripper - (cube + np.array([0, 0, 0.06]))) < 0.06:
        holding = True

    if holding:
        cube = gripper - np.array([0, 0, 0.06])

    if holding and np.linalg.norm(cube[:2] - target[:2]) < 0.04 and cube[2] < 0.12:
        holding = False

    data.mocap_pos[0] = gripper
    set_cube_pos(data, cube)
    mujoco.mj_forward(model, data)

    renderer.update_scene(data, camera="cam")
    frames.append(renderer.render())

imageio.mimsave("bc_pickplace.mp4", frames, fps=30)
print("Saved bc_pickplace.mp4")

The expert action is the hand-coded demonstration signal for moving from the cube to the green circle. The behaviour cloning model learns directly from these state-action examples. 

loss_fn = nn.MSELoss()

In this quick demonstration, mean squared error penalizes the difference between the predicted action and the demonstrated expert action.

Note: Even though this technique as a standalone method can struggle on complex tasks, it is often useful for pretraining or warm-starting reinforcement learning.
